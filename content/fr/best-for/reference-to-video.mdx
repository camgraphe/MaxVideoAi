## Ce que vous cherchez
Le reference-to-video est plus large que le character reference. La référence peut être une image produit, une vidéo existante, une ambiance, une piste audio ou une direction de campagne. Le but est de guider le résultat, pas toujours de conserver une identité exacte.

## Meilleurs choix
1. **[Happy Horse 1.0](/fr/modeles/happy-horse-1-0)** - meilleur choix global pour R2V, image de départ, édition V2V, audio natif et lip-sync.
2. **[Seedance 2.0](/fr/modeles/seedance-2-0)** - excellent pour workflows de références multimodales.
3. **[Veo 3.1](/fr/modeles/veo-3-1)** - très bon pour un plan propre à partir d’une référence image.
4. **[Kling 3 Pro](/fr/modeles/kling-3-pro)** - meilleur quand les références doivent devenir une séquence structurée.

## Pourquoi Happy Horse est premier
Happy Horse 1.0 est le meilleur point de départ quand vous arrivez avec du matériel de référence et que le workflow peut ensuite demander image-to-video, V2V, audio natif ou lip-sync. Son mode R2V gère plusieurs images de référence, sans forcer un modèle différent pour l’étape suivante.

Seedance 2.0 reste excellent pour les références multimodales. Kling 3 Pro devient meilleur lorsque la référence doit rester cohérente sur plusieurs plans.

## FAQ
### Quelle différence entre image-to-video et reference-to-video ?
L’image-to-video anime souvent une seule image. Le reference-to-video utilise un ensemble plus large de références pour guider la sortie.
