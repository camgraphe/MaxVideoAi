## Ce que vous cherchez
Le character reference est plus strict qu’une simple référence visuelle. Vous demandez au modèle de conserver identité, silhouette, vêtements, props et style malgré le mouvement caméra. C’est pour cela que cette page ne recommande pas les mêmes modèles qu’une page reference-to-video générale.

## Meilleurs choix
1. **[Happy Horse 1.0](/fr/modeles/happy-horse-1-0)** - meilleur choix global quand les références personnage doivent aussi garder audio natif, lip-sync et édition V2V.
2. **[Kling 3 Pro](/fr/modeles/kling-3-pro)** - meilleur pour la continuité personnage/prop avec Elements, multi-shot et contrôle scène.
3. **[Seedance 2.0](/fr/modeles/seedance-2-0)** - excellent quand l’identité du personnage fait partie d’un pack de références plus large.
4. **[Kling 3 Standard](/fr/modeles/kling-3-standard)** - meilleur choix value pour tester la continuité.

## Pourquoi Happy Horse mène
Happy Horse 1.0 mène ici parce que son mode R2V accepte plusieurs images de référence et reste connecté aux modes image-to-video, text-to-video et V2V. C’est utile quand un personnage doit aussi parler, garder une ambiance audio native ou subir un edit visuel après génération.

## Pourquoi Kling 3 mène
Kling 3 Pro et Standard permettent de structurer les scènes et de réutiliser des Elements comme anchors. C’est plus robuste que répéter une longue description de personnage dans chaque phrase du prompt. Seedance 2.0 reste pertinent quand le personnage fait partie d’une direction de campagne plus large.

## FAQ
### Quel modèle choisir pour un personnage de référence ?
Happy Horse 1.0 est le meilleur point de départ si la référence doit aussi couvrir audio, lip-sync ou édition V2V. Kling 3 Pro reste l’alternative la plus forte pour les scènes multi-shot avec Elements.
